The Problem

Text-based guardrails can catch what an AI says, but they miss what it does. An agent that never outputs harmful content can still call delete_records(record_count=500) or send_email(to="competitor@example.com") — and by the time you see the chat log, the damage is done.

Action Guardrail sits between the agent and its tools. Every tool call is evaluated against a set of declarative rules before execution. No false promises about "responsible AI" — just a programmable policy layer that enforces what your infrastructure already knows.

How It Works

Four steps, zero friction.

1

Agent Proposes

The AI agent calls a tool — e.g. delete_records — with parameters.

→
2

Policy Evaluates

Guardrail matches the call against your YAML rules by tool name and conditions.

→
3

Decision Made

First matching rule wins: block, require_hitl, or log_and_allow.

→
4

Audit Logged

Every decision is recorded with a correlation ID for forensics and compliance.

See It In Action

These three real scenarios ship as the default rule set. Open the dashboard with the key AryaGuardrail1804 to see live results.

Bulk Delete Blocked

delete_records({ "count": 500 }) block

External Email Held

send_email({ "to": "external@example.com" }) require_hitl

Confidential Read Logged

read_file({ "path": "/data/confidential/report.pdf" }) log_and_allow
